# Usuarios y equipos en Linux

## Acceso como superusuario
Acceder al usuario como *root* en linux puede acarrear varios problemas de seguridad, y líos con las identidades de las personas que entran o dejan de entrar (haciendo imposible llevar un historial). Como solución a esto existe el comando `su -` con el que, haciendo uso de las propiedades de setuid, y aunque no se guarda un registro de sus acciones como root, pero sí quién accedió como superusuario a través de dicho comando y cuándo, lo que puede ser almacenado en los registros de un fichero log.

`"su -" == "su - root"` son equivalentes.

Aunque en algunos sistemas ambas medidas se encuentras bloqueadas, por lo que se recomienda hacer uso del comando [[sudo]], el cual también toma ventaja de las propiedades de setuid. 
# Cuentas de usuarios
## Creación de usuarios y fichero /etc/passwd
Para almacenar los username y los id de cada usuario, es necesario una base de datos, denominada /etc/passwd. 

Cada entrada se corresponde a un usuario y se oresenta en el siguiente formato:
`username:x:UID:GID:user-information:home-directory:login-shell`

donde cada campo está separado por “:”.
